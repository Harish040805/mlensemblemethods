How to Use the Simulator

A step-by-step user guide for navigating and operating the interface.

How to Navigate the Site

Follow these structural steps to move through the simulator interface:

  1. Click the Enter the site button on the home page to open the simulation selection screen.
  2. Select either the Bagging and Boosting option or the Boosting Types option.
  3. Once inside a dashboard, click the horizontal Tabs at the top of the panel to switch the active workspace view.

Where and How to Give Inputs

Interactive Triggers: Inputs are provided entirely through explicit action buttons. There are no text boxes or configuration forms to fill out manually.

  • Bagging View: Click the button labeled "Run Bagging Step" to generate the initial data layouts and run the setup.
  • Boosting View: Click the button labeled "Run Next Sequential Round" to alter the interface state.
  • AdaBoost View: Click the button labeled "Click to Fix Mistake" to progress the layout.
  • XGBoost View: Click the button labeled "Click to Correct Error" to advance the step.

Where to See Output

The interface updates and displays results in two distinct areas on the screen:

  • The Visual Arena: Located in the center of the panel. You will see graphical shifts here, such as elements changing scale size, text indicators updating, or color blocks altering instantly upon clicking an action button.
  • The Output Display Bar: Located at the absolute bottom of each panel. It provides a running textual summary statement regarding the final consolidated state of the simulation.

Which is a Correct Output?

Completion Criteria: The interface indicates a simulation run has successfully completed when specific terminal states are reached.

  • The action control buttons become greyed out and unclickable.
  • The bottom summary text transitions to a final green status message.
  • The progress bar fills completely or shifts down to 0% depending on the active simulator dashboard metrics.
